What this tool is

        

          An internal, localhost-only UI for signing DocRouter product licenses
          (DRLIC1.…) and inspecting existing tokens. It is not part of the
          customer product. The signing private key stays on your machine.
        


        
One-time key setup

        
Generate an Ed25519 keypair (from the repo root, venv active):

        
python scripts/licensing/generate_keys.py --out-dir /tmp/dr-lic
mkdir -p ~/.ssh
cp /tmp/dr-lic/license-private.pem ~/.ssh/docrouter-license-private.pem
chmod 600 ~/.ssh/docrouter-license-private.pem
cp /tmp/dr-lic/license-public.pem \
  packages/python/analytiq_data/licensing/keys/license-public.pem

        

          Commit only the public key with the product. Never commit the private key.
          Customer deployments verify tokens with the packaged public key.
        


        
Start this UI

        
python scripts/licensing/manage_ui.py
# open http://127.0.0.1:8765

        

          Defaults to ~/.ssh/docrouter-license-private.pem.
          Override with --private-key / --password if needed.
        


        
Generate a license

        
    
          
  1. Open the Generate tab.
    2. 
          
  2. Fill customer fields, start date, duration, and features (documents, flows).
    3. 
          
  3. Start date may be past or future; prefer MM/DD/YYYY (e.g. 7/1/2024) or YYYY-MM-DD.
    4. 
          
  4. Optionally bind to an installation ID from the customer’s Settings → License page.
    5. 
          
  5. Click Generate license and copy the DRLIC1.… token.
    6. 
        

        

          CLI equivalent:
        

        
python scripts/licensing/issue_license.py \
  --claims claims.json \
  --private-key ~/.ssh/docrouter-license-private.pem


        
Install / view a license (customer product)

        
    
          
  1. Sign in as a system admin.
    2. 
          
  2. Go to Settings → System → License.
    3. 
          
  3. Paste the token and save. Status shows customer, expiry, features, and installation ID.
    4. 
          
  4. Or bootstrap on first start with LICENSE_KEY / LICENSE_FILE (see docs/env.md).
    5. 
        

        

          No key in Mongo → product is ungated (SaaS / fresh install). Installing a key turns
          enforcement on. Expired/invalid keys disable most of the HTTP API but still allow
          auth and this License settings page so admins can upload a replacement.
        


        
Review a license

        

          Use the Review tab to paste any DRLIC1 token. Verification
          uses the public key derived from your local private key (so it must match the key that
          signed the token). You will see decoded claims plus mode
          (licensed / grace / expired / invalid).
